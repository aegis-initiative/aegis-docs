# Guides

These guides walk through practical tasks and common patterns for working with the AEGIS governance platform. They are designed to be followed step by step and build on the concepts introduced in [Getting Started](/getting-started/).

## Available Guides

### [Policy Authoring](/guides/policy-authoring/)

Learn how to write governance policies that define what AI agents can and cannot do. Covers capability registration, policy rules, conditional logic, and testing policies before deployment.

### [Deployment](/guides/deployment/)

Deploy the AEGIS governance runtime in your infrastructure. Covers self-hosted deployment options, managed platform setup, and integration with existing CI/CD pipelines.

### [Monitoring](/guides/monitoring/)

Monitor governance activity in real time. Covers the operator dashboard, audit log queries, alerting on governance events, and building compliance reports.

### [Troubleshooting](/guides/troubleshooting/)

Diagnose and resolve common issues with AEGIS integrations. Covers connection problems, unexpected denials, policy debugging, and performance optimization.

## Prerequisites

Before following these guides, you should be familiar with:

- [Core Concepts](/getting-started/core-concepts/) -- The governance model and its components
- [Quick Start](/getting-started/quick-start/) -- Basic SDK setup and usage

> **Note:** These guides are evolving alongside the AEGIS platform. Content will be expanded as new features and deployment patterns are established.
