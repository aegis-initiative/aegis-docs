## Development Log

- Updated the org context headers to align with the canonical legal entity structure: Operating Entity AEGIS Operations LLC, Trademark Owner Finnoybu IP LLC (#23)
- Added `public/_headers` to pin Cloudflare Pages response headers per path — long TTLs for fingerprinted and immutable assets (`/_astro/*`, `/fonts/*`), moderate TTLs for brand assets, `max-age=0, must-revalidate` for HTML (#24)
- Added a committed `VERSION` file at the repo root as the machine-readable source of truth for the current release — read by the Astro build at init time to populate `import.meta.env.AEGIS_VERSION` for the header badge (#24)
- Migrated the inline `resolveVersion()` helper in `astro.config.mjs` to a two-line import of `readSiteVersion()` from `@aegis-initiative/design-system/build`, published as v0.4.0 (#25)
- Fixed the "Try it now" call-to-action on the landing page — the previous link pointed at a no-longer-available API playground at `demo.aegis-platform.net/docs`; now points at the live operator dashboard at `demo.aegis-platform.net/` (#26)
